Inputs HTML5: ¿Y si no están soportados por el navegador?

No todos los nuevos input estan soportados por los navegadores. Por ejemplo, date, time o color. Si no están soportados por el navegador, este ignora el tipo y considera la entrada de tipo texto.

En estos casos existe la posibilidad de comprobar si el navegador soporta el tipo (con Modernizr por ejemplo), y si no lo hace añadir una solución "programada" (por ejemplo JQuery da la misma funcionalidad con el método datepicker).

En esta página se utiliza el <input type="date"> que a fecha 01-10-2016 no está soportado por Firefox. En caso de mostrar la página con Chrome, se muestra el input nativo HTML5. En caso de usar Firefox se detecta que no lo soporta, y en ese caso se seleccionan los input de la clase fecha y se les aplica el método datepicker de jQueryUI.

En general se habla en estos casos de Polyfill: código o plugin que permite obtener las funcionalidades de HTML5 en navegadores que no lo soportan.

Formulario de ejemplo con input de tipo date

En este ejemplo se ha añadido en la cabecera código similar a este, con una llamada a alert que le indica si el tipo date esta soportado o no:


<script type="text/javascript">
$(function(){
  if(!Modernizr.inputtypes.date) { // Comprueba si está soportado
    alert("tipo date no soportado. Se simula ese tipo mediante polyfill con JQueryUI");
    console.log("The 'date' input type is not supported, so using JQueryUI datepicker instead.");
    $("input.fecha").datepicker(); // selecciona todos los class fecha y le asocia el calendario de JQueryUI
  }	
}); 
</script>
